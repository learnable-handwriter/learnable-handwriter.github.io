Annotation Format:

The annotation.json file must contain entries in this exact format:

{
  "<image_id>": {
    "split": "train",                    # {"train", "val", "test"}
    "label": "your transcription text",  # The ground truth text
    "script": "Script_Type_Name",        # (optional) Script classification
    "page": "manuscript_page.jpg"        # (optional) Source page reference
  },
  ...
}
Important: All fields are necessary except "page" and "script". For unsupervised training without evaluation, you can omit the entire annotation.json file.

ğŸ”¬ Scientific Guidelines & Best Practices

Following these evidence-based guidelines will significantly improve your model's performance and training stability:

âœï¸ Script Articulation: Prioritize articulated scripts where individual letters are well-separated over highly cursive writing. While cursive scripts can be tested, clear letter boundaries improve character recognition accuracy by 15-25% in our experiments.
âœ‚ï¸ Precise Segmentation: Ensure accurate polygon segmentation with no surrounding letters or extraneous marks. Clean boundaries are critical for proper character isolation.
Left: Clean segmentation. Right: Poor segmentation with surrounding letters
ğŸ“ Text Orientation: Minimize text inclination whenever possible. Images should be as horizontally aligned as feasible. Excessive skew (>5Â°) can reduce recognition accuracy and increase training time.
ğŸ¨ Alpha Channel: Always include the alpha channel in your images. This transparency information helps the model distinguish between ink and background more effectively.
ğŸš« Image Quality: Avoid images with ink spills, severe parchment damage, or excessive noise. These artifacts can confuse the training process and reduce model generalization.
ğŸ“ Consistent Line Length: Maintain coherent line lengths throughout your dataset. Avoid mixing very short lines (e.g., from column layouts) with very long lines (e.g., from charters). Inconsistent lengths can cause memory allocation issues during training.
ğŸŒ“ High Contrast: Ensure strong contrast between ink and background. Poor contrast significantly impacts character recognition performance.
Top: Good contrast. Bottom: Poor contrast affecting recognition
ğŸ“Š Character Distribution: Include at least 30 characters per class, with >50 characters being optimal. Insufficient character examples lead to poor generalization and class imbalance issues. This is based on statistical learning theory requirements for reliable pattern recognition.
âš ï¸ Critical Success Factors: The most common causes of training failure are: (1) inconsistent line lengths causing memory errors, (2) insufficient character examples per class, and (3) poor image contrast. Address these first before troubleshooting other issues.

ğŸ¯ Fine-tuning Options

1. Script-based Fine-tuning (Northern and Southern Textualis):

  -o runs/iwcp_south_north/finetune/ \

  -o runs/iwcp_south_north/finetune/ \
  --mode g_theta --max_steps 2500 --invert_sprites \
  -a datasets/iwcp_south_north/annotation.json \
  -d datasets/iwcp_south_north/ --split all

ğŸ“ Training on Your Custom Data

Configuration Setup

  1. Create dataset config: configs/dataset/<DATASET_ID>.yaml 
    DATASET-TAG:                 
  path: <DATASET-NAME>/      
  sep: ''                    # Character separator in annotation
  space: ' '                 # Space representation in annotation
  2. Create hyperparameter config: configs/<DATASET_ID>.yaml
    For structure reference, see the provided config file for our iwcp_south_north experiment.

Training Process

  1. Initial Training: 
    python scripts/train.py <CONFIG_NAME>.yaml
  2. Fine-tuning by Script Type: 
    python scripts/finetune_scripts.py -i runs/<MODEL_PATH>/ \
  -o <OUTPUT_PATH>/ --mode g_theta --max_steps <int> \
  --invert_sprites --script '<SCRIPT_NAME>' \
  -a <DATASET_PATH>/annotation.json \
  -d <DATASET_PATH>/ --split <train or all>
  3. Fine-tuning by Individual Documents: 
    python scripts/finetune_docs.py -i runs/<MODEL_PATH>/ \
  -o <OUTPUT_PATH>/ --mode g_theta --max_steps <int> \
  --invert_sprites -a <DATASET_PATH>/annotation.json \
  -d <DATASET_PATH>/ --split <train or all>
